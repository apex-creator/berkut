About Spectrum based feature extraction

The feature extraction based on specrtum computes features generated out of micro-doppler spectrum which is created from each frame of data. Currently the feature extraction includes the below features from micro-doppler spectrum heatmap:

[1] Lower frequency border of the occupied bandwidth.
[2] Upper frequency border of the occupied bandwidth.
[3] Power within the occupied bandwidth
[4] Mean frequency of the power spectral density estimate
[5] Median frequency of the power spectral density estimate,.
[6] Spectral entropy of the power spectral density estimate

(
FeatExtract_moduleConfig *config,
int32_t *errCode
)

Description: Feature extraction create function. It allocates memory to store its internal data object, assign the config to internal parameters and returns a handle if it executes successfully.
Parameters:
initCfg – config Pointer to initial configuration parameter.
errCode – errCode Pointer to errCode generates by the API.

Return values:
Success = valid handle
Error != 0: NULL

)

Description: Feature extraction computation. It computes the features from micro-doppler spectrum using featExtract_computeSpectralEntropy function, featExtract_computeWeightedPower along with other helper functions for math and specific utility.

handle – featExtract handle.

featOut – Pointer to output extracted features.
